# 문제은행시스템 UI/UX 스타일 가이드 v1.1

본 문서는 **문제은행시스템(관리자/위원)** 웹 화면의 **UI/UX 일관성**을 위한 스타일 가이드이다.  
기술스택(프론트: Vue+Vite)은 구현 수단이며, 본 가이드는 **화면/컴포넌트의 시각·인터랙션 규칙**을 정의한다.

---

## 1. 적용 범위

- 대상 화면
  - 공통: 로그인, 마이페이지
  - 관리자: 출제/검토/선정/시험지/문항(콘텐츠)/시스템관리
  - 위원: 출제하기/검토하기/선정하기
- 공통 레이아웃 패턴
  - **좌측 트리 + 우측 검색/목록 + 상세 패널(또는 하단 상세)** 구조
  - 목록 중심 화면의 표준 검색 영역/표준 테이블/표준 상세 탭

---

## 2. 디자인 원칙

1) **명확성(Clarity)**: 상태/권한/마감/잠금 등 업무 핵심 정보가 한눈에 보이도록 한다.  
2) **일관성(Consistency)**: 같은 개념은 같은 위치·표현·용어·색을 사용한다.  
3) **효율성(Efficiency)**: 목록 중심 업무는 키보드/빠른 필터/일괄 처리로 동선을 줄인다.  
4) **안전성(Safety)**: 확정/마감/삭제/내보내기 등 위험 행동은 확인·권한·로그를 포함한다.  
5) **접근성(Accessibility)**: 대비/포커스/키보드/스크린리더를 기본 제공한다.

---

## 3. 레이아웃/그리드

### 3.1 화면 기본 구조
- 상단: 글로벌 헤더(시스템명, 사용자 메뉴, 알림/도움말(옵션))
- 좌측: 사이드 메뉴(권한 기반 노출) 또는 트리 패널(문항/분류 중심 화면)
- 본문: 페이지 타이틀 + 설명(옵션) + 컨텐츠

### 3.2 좌 트리 / 우 목록 / 상세 패턴(표준)
- 좌측 트리 폭: 280~360px 권장(리사이즈 가능하면 옵션)
- 우측 영역
  - 상단: 검색조건(필터 바)
  - 중단: 테이블 목록
  - 우측 또는 하단: 상세 패널(탭형)

### 3.3 여백/간격(Spacing)
- 4px 단위 스케일 권장: 4 / 8 / 12 / 16 / 20 / 24 / 32
- 화면 섹션 간 간격: 16~24
- 카드/패널 내부 패딩: 16(기본), 24(상세)

### 3.4 반응형(권장 브레이크포인트)
- 1440px 이하: 사이드 여백 축소, 테이블 컬럼 줄임/접기
- 1024px 이하: 상세 패널은 Drawer(우측 슬라이드)로 전환 권장
- 768px 이하(위원 업무 최소 지원): 트리는 상단 토글/모달로 전환, 테이블은 카드형 요약 가능

---

## 4. 타이포그래피

### 4.1 폰트
- 기본: 시스템 기본 한글 폰트(운영 표준에 맞춰 지정)
- 숫자/코드 가독성: 탭 정렬/고정폭 스타일은 선택적으로 적용

### 4.2 폰트 크기/굵기(권장)
- Page Title: 20~24 / Bold
- Section Title: 16~18 / SemiBold
- Body: 14~15 / Regular
- Caption/Hint: 12~13 / Regular
- 테이블 기본: 13~14

### 4.3 텍스트 규칙
- 라벨은 짧고 명확하게(동사/명사 통일)
- 긴 설명은 도움말 아이콘/툴팁/접기 제공
- 숫자/단위는 붙여쓰기 규칙 통일(예: 10건, 3초)

---

## 5. 컬러/상태 표현(토큰 방식 권장)

색상은 브랜드/기관 가이드가 있으면 그 체계에 맞춰 토큰만 매핑한다.

### 5.1 기본 토큰
- Primary: 주요 액션(저장/제출/확정)
- Secondary: 보조 액션(검색/적용/닫기)
- Neutral: 기본 텍스트/보더/배경
- Danger: 삭제/회수/강제 종료/치명 오류
- Warning: 마감 임박/주의 필요/잠재 위험
- Success: 완료/통과/성공
- Info: 안내/참고

### 5.2 상태 배지(필수)
- 계획: 작성중 / 확정 / 마감
- 의뢰: 미발송 / 발송 / 진행중 / 마감 / 회수(정책)
- 수행: 미작성 / 임시저장 / 제출 / 반려 / 완료
- 배지는 **텍스트 + 색상**을 함께 사용하고, 색만으로 의미 전달하지 않는다.

---

## 6. 아이콘/표기 규칙

- 아이콘은 의미를 보조(단독 사용 금지, 툴팁/라벨 병기)
- 트리 노드:
  - 기본: 폴더/문서 아이콘
  - 비활성: 흐린 처리 + 비활성 배지
- 상태 아이콘:
  - 성공/실패/진행/잠금은 일관된 아이콘 세트 사용

---

## 7. 컴포넌트 스타일 규칙

## 7.1 버튼(Button)
- 크기: Sm / Md(기본) / Lg
- 종류:
  - Primary: 저장/제출/확정/발송 등 주요 행동(한 화면 1개 권장)
  - Secondary: 검색/적용/닫기
  - Tertiary(텍스트): 상세보기/링크/보조 기능
  - Danger: 삭제/회수/폐기(확인 모달 필수)
- 비활성(Disabled): 클릭 불가 + 툴팁(사유 제공 권장)

## 7.2 입력/폼(Form)
- 라벨은 좌측 상단 정렬(필수 표시는 * 또는 (필수) 통일)
- 검증 메시지: 입력 하단 1줄(요약), 상세는 툴팁/팝오버
- 날짜/기간: 시작~종료 일관 컴포넌트 사용
- Select/Combo: 검색 가능한 옵션이 많으면 검색형 콤보 사용

## 7.3 테이블(Table)
- 기본: 헤더 고정(옵션), 행 hover, 정렬, 페이징
- 컬럼 우선순위
  - 핵심 식별(코드/명) > 상태/마감 > 담당자 > 날짜/시간 > 기타
- 빈 상태(Empty)
  - “조건에 맞는 데이터가 없습니다.” + 초기화 버튼
- 로딩(Loading)
  - 스켈레톤 또는 로딩 오버레이(테이블 영역만)

## 7.4 트리(Tree)
- 검색: 트리 상단 검색 입력 + 결과 하이라이트 + 다음/이전 이동
- “하위 포함” 옵션은 우측 검색 영역에 표준으로 제공
- 비활성 노드: 선택 제한(신규 등록 시) + 표시 구분

## 7.5 상세 패널(Detail)
- 탭 구성 권장: 기본정보 / 본문 / 보기 / 정답·해설 / 첨부 / 이력
- 저장되지 않은 변경이 있을 때:
  - 탭 이동/행 이동 시 경고(저장/폐기 선택)
- 읽기/편집 모드:
  - 상세는 기본 읽기, 편집은 명시적 전환(버튼) 권장

## 7.6 모달/다이얼로그(Modal)
- 사용 기준: 삭제/확정/마감/회수/내보내기 등 위험 행동
- 구조: 제목(행동 요약) + 본문(영향) + 확인/취소(오른쪽 정렬)
- ESC 닫힘 정책은 중요도에 따라 선택(삭제/확정은 ESC 금지 권장)

## 7.7 알림/토스트(Toast)
- Success: 짧게(예: “저장되었습니다.”)
- Error: 사용자 메시지(원인/조치) + 필요 시 상세 보기(에러코드/요청ID)
- Warning: 마감 임박/권한 부족/잠금 상태 안내

## 7.8 파일 업로드(File)
- 업로드 영역: 파일명/용량/확장자 안내
- 진행 상태: 업로드 중 표시 + 취소(옵션)
- 다운로드: 권한 체크 실패 시 명확한 안내

---

## 8. 화면 패턴(업무 시나리오)

### 8.1 계획/의뢰/현황 공통
- 상단: 계획/의뢰 기본 정보(기간, 상태, 담당)
- 중앙: 배정 목록(위원/분야/진행률)
- 하단: 대상 문항 목록(드릴다운)

### 8.2 콘텐츠(content) 통합 관리(quiz/test)
- 상단 탭 또는 좌 메뉴로 문항/시험지 전환
- 시험지 편성 화면에서는 문항 검색 패널(분류/필터)을 재사용
- 문항 상세에서 “포함된 시험지 목록” 링크 제공(탐색성 강화)

### 8.3 확정/마감/잠금 UX
- 확정/마감 상태에서는 입력 컨트롤 비활성
- 비활성 사유를 UI에 표시(“확정본 잠금”, “마감 완료”)
- 잠금 해제 정책이 있다면 관리자 전용 경로로 분리

---

## 9. 접근성/품질 체크리스트

- 대비(텍스트/배경) 기준 충족
- 키보드 내비게이션:
  - Tab 이동 순서 자연스러움
  - 포커스 링 표시
- 스크린리더:
  - 버튼/아이콘에 aria-label 제공
- 오류 메시지:
  - 입력 항목과 연결되어야 함
- 성능:
  - 테이블은 가상 스크롤/지연 로딩 고려(대량 데이터)

---

## 10. 용어/표기 통일(권장)
- 계획: Plan
- 의뢰: Request/Assign
- 위원: Committee Member
- 문항: Quiz/Question (내부 용어는 한 가지로 통일)
- 시험지: Test Paper
- “저장/임시저장/제출/확정/마감/회수” 버튼 라벨은 전 화면에서 동일하게 사용
